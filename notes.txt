Project:
--------
bias-ai-recruitment/

Core idea:
----------
This project is a controlled pipeline for studying bias in LLM-based recruitment evaluation.

We generate synthetic CV templates, derive tightly matched variants where only one target
attribute should differ, build head-to-head evaluation sessions, ask an LLM to compare
the candidates, and then analyze whether the model systematically favors one variant value
over another.

The point is not only to generate CVs, but to maintain an experimental structure where
differences in outcomes can be interpreted in a controlled way.

What each main script does:
---------------------------
src/generator.py
- Generates synthetic base CV templates
- Generates controlled variants from those templates

src/session_builder.py
- Builds evaluation sessions from matched candidate pairs
- Creates one or more replicated comparisons per prompt condition

src/evaluator.py
- Sends candidate comparison prompts to the target LLM provider
- Stores raw outputs, parsed outputs, and line-level results

src/analysis.py
- Filters results against the current sessions and current variants
- Computes aggregated metrics
- Saves charts and metrics

src/preview.py
- Renders a candidate preview

src/session_preview.py
- Renders a session preview, useful for checking comparisons before evaluation

src/run_pipeline.py
- Orchestrates the end-to-end pipeline
- Can also skip session building, evaluation, or analysis when needed

Environment setup:
------------------
Create a virtual environment and install dependencies:

python -m venv .venv
source .venv/bin/activate

Windows:
.venv\Scripts\activate

Install dependencies:
pip install -r requirements.txt

API keys:
---------
Create a .env file in the project root:

OPENAI_API_KEY=your_openai_key
MISTRAL_API_KEY=your_mistral_key

Only the provider you actually use needs to be configured.

Main commands:
--------------

1) Full pipeline:
python -m src.run_pipeline \
  --job "Data Analyst" \
  --languages English \
  --template-count 10 \
  --provider openai \
  --model gpt-4o-mini \
  --scenario single_attribute \
  --prompts neutral_v1 strict_merit_v1 role_specific_v1 \
  --replicates 3 \
  --concurrency 5 \
  --reset

2) Generate CVs and variants only, without evaluation:
python -m src.run_pipeline \
  --job "Data Analyst" \
  --languages English \
  --template-count 10 \
  --provider openai \
  --model gpt-4o-mini \
  --scenario single_attribute \
  --prompts neutral_v1 strict_merit_v1 role_specific_v1 \
  --replicates 3 \
  --concurrency 5 \
  --reset \
  --skip-evaluation \
  --skip-analysis

3) Generate only templates:
python -m src.generator templates \
  --job "Data Analyst" \
  --language English \
  --count 10 \
  --provider openai \
  --model gpt-4o-mini \
  --reset

4) Generate only variants:
python -m src.generator variants \
  --provider openai \
  --model gpt-4o-mini \
  --scenario single_attribute \
  --reset

5) Build sessions:
python -m src.session_builder \
  --prompts neutral_v1 strict_merit_v1 role_specific_v1 \
  --replicates 3

6) Evaluate sessions:
python -m src.evaluator \
  --provider openai \
  --model gpt-4o-mini \
  --concurrency 5

7) Analyze results:
python -m src.analysis

Important implementation notes:
-------------------------------
- The pipeline now supports generation without forced evaluation
- The main reset logic is intended to clear both data and outputs
- Analysis should be run on the current dataset, not on stale historical leftovers
- Evaluation prompts are experimental conditions, not duplicate files
- If scores become suspiciously repetitive again, inspect the evaluation prompts first
- If results reference CV IDs missing from variants.jsonl, the first thing to check is whether an old run was mixed into the current state

About evaluate.txt:
-------------------
The evaluator currently uses prompts named:
prompts/evaluate_<prompt_id>.txt

That means the active prompt files are:
- evaluate_neutral_v1.txt
- evaluate_strict_merit_v1.txt
- evaluate_role_specific_v1.txt

If evaluate.txt is not used anywhere in the workflow, it can be removed later to reduce confusion.
