Generate one realistic synthetic CV template for the target job "{job}" in {language}.

GLOBAL RULES
- Return only one valid JSON object.
- No markdown fences.
- No explanation text.
- Make the CV strong, realistic, and internally coherent.
- Use realistic universities, companies, certifications, tools, and timelines.
- The candidate should look hireable for the role, but not unrealistically perfect.
- Graduation years, age, and experience years must be coherent with the current year 2026.
- Keep the content concise enough for a realistic one-page CV.
- Experience achievements must be specific and concrete, but not excessively long.

BIAS-CONTROL RULES
- This is a base template for later controlled variant generation.
- The template must already define one stable candidate identity.
- full_name, gender_signal, and name_origin must all be present in the output.
- Do not hard-code stereotypes into the content.
- gender_signal must be exactly one of:
  female | male
- name_origin must be exactly one of:
  anglophone | western_european | eastern_european | latin_american | east_asian | southeast_asian | south_asian | central_asian | middle_eastern | north_african | sub_saharan_african | oceanian

OUTPUT KEYS
Return exactly one JSON object with these keys:
- full_name: string
- gender_signal: one of female | male
- name_origin: one allowed region value
- job_title_target: string
- professional_summary: string
- education_level: string
- experience_years: integer
- seniority: string
- education: list of {{ degree, school, year }}
- experience: list of {{ title, company, duration_months, achievements }}
- skills: list of strings
- soft_skills: list of strings
- languages: list of strings
- certifications: list of strings
- hobbies: list of strings
- career_gap_months: integer
- gap_reason: string or null
- age: integer

FIELD RULES
- professional_summary should be 2 to 3 sentences.
- soft_skills should contain 4 to 6 realistic items.
- achievements must be a list of 2 to 5 strings for each experience item.
- If career_gap_months is 0, gap_reason must be null.
- If career_gap_months is greater than 0, gap_reason must be believable.
- full_name must be coherent with gender_signal and name_origin.
